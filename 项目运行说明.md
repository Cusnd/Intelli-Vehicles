# 🚗 智能车辆维保记录管理系统 - 运行说明

## 项目概述
这是一个完整的车辆维保记录管理系统，包含前端Vue3应用和后端Spring Boot服务。

## 技术栈
### 后端
- Spring Boot 3.x
- MySQL 8.0
- MyBatis-Plus
- Spring Security + JWT
- Maven

### 前端
- Vue 3
- Element Plus
- Vue Router
- Axios
- Vite

## API 接口

### 认证相关
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册
- `GET /api/auth/profile` - 获取用户信息

### 车辆管理
- `GET /api/vehicles` - 获取车辆列表
- `GET /api/vehicles/{id}` - 获取车辆详情
- `POST /api/vehicles` - 添加车辆
- `PUT /api/vehicles/{id}` - 更新车辆
- `DELETE /api/vehicles/{id}` - 删除车辆

### 操作日志
- `GET /api/logs` - 获取日志列表
- `POST /api/logs` - 记录日志
- `DELETE /api/logs/cleanup` - 清理日志

## 开发说明

### 添加新功能
1. 后端：在对应的controller、service、mapper中添加代码
2. 前端：在pages目录下创建新页面，在api目录下添加接口调用
3. 更新路由配置

### 数据库配置
修改 `backend/vehicles/src/main/resources/application.yml` 中的数据库连接信息：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/vehicle_maintenance?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=GMT%2B8&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
    username: your_username
    password: your_password
```

## 常见问题

### 1. 数据库连接失败
- 检查MySQL服务是否启动
- 确认数据库连接信息是否正确
- 确保数据库已创建

### 2. 前端无法访问后端API
- 检查后端服务是否正常启动（端口8081）
- 检查vite.config.js中的代理配置

### 3. 登录失败
- 确认数据库中是否有用户数据
- 检查JWT密钥配置
